# mysqlcli 设计文档

## 1. 文档信息

- 项目名称：`mysqlcli`
- 文档状态：实现中
- 目标平台：macOS、Linux，后续评估 Windows
- 目标数据库：MySQL 5.7、MySQL 8.x，兼容性验证 MariaDB
- 实现语言：Go

## 2. 项目背景

`mysqlcli` 是一个面向开发和数据库运维场景的 MySQL 命令行客户端。它保持官方 `mysql` 客户端常用连接参数和使用习惯，同时提供类似 PostgreSQL `psql` 的元数据、会话、锁等待、复制、用户及权限快捷命令。

项目重点不是一次性复制官方客户端全部功能，而是提供一个启动快、易分发、查询结果清晰、诊断操作高效的现代终端工具。

## 3. 设计目标

1. 常见官方 `mysql` 连接命令可低成本替换为 `mysqlcli`。
2. 支持交互式 SQL、多行输入、脚本执行和非交互执行。
3. 使用统一快捷命令查看数据库对象和常用运维信息。
4. 自动适配 MySQL 5.7 与 8.x 的元数据差异。
5. 默认保护密码、认证信息及历史记录中的敏感数据。
6. 输出既适合人工阅读，也支持脚本消费。

## 4. 非目标

首个版本不包含以下内容：

- 完整兼容官方 `mysql` 的全部冷门参数和所有反斜杠命令。
- 替代专业监控平台、审计系统或复制管理平台。
- 数据库管理图形界面。
- 在客户端内实现 MySQL 协议或 SQL 解析器的完整语法树。
- 自动修复锁、复制或权限问题。

## 5. 用户场景

### 5.1 交互连接

```shell
mysqlcli -h 127.0.0.1 -P 3306 -u root -p app
mysqlcli --socket /tmp/mysql.sock -u root -p
```

### 5.2 非交互查询

```shell
mysqlcli -h 127.0.0.1 -u root -p -D app -e "SELECT VERSION()"
mysqlcli app < schema.sql
```

### 5.3 运维诊断

```text
\sessions
\locks
\repl
\variables innodb
\warnings
\du
\grants app_user@localhost
```

## 6. 命令行兼容设计

### 6.1 首版支持参数

| 短参数 | 长参数 | 说明 |
| --- | --- | --- |
| `-h` | `--host` | MySQL 主机地址 |
| `-P` | `--port` | MySQL TCP 端口 |
| `-u` | `--user` | 登录用户 |
| `-p[密码]` | `--password[=密码]` | 不提供值时安全读取密码 |
| `-D` | `--database` | 初始数据库 |
| `-S` | `--socket` | Unix Socket 路径 |
| `-e` | `--execute` | 执行 SQL 后退出 |
| `-B` | `--batch` | 批处理输出 |
| `-N` | `--skip-column-names` | 不输出列名 |
| `-s` | `--silent` | 精简输出 |
| 无 | `--protocol` | `tcp` 或 `socket` |
| 无 | `--connect-timeout` | 连接超时 |
| 无 | `--ssl-mode` | TLS 校验模式 |
| 无 | `--ssl-ca` | CA 文件 |
| 无 | `--ssl-cert` | 客户端证书 |
| 无 | `--ssl-key` | 客户端私钥 |
| 无 | `--defaults-file` | 指定客户端配置文件 |
| 无 | `--defaults-extra-file` | 加载附加配置文件 |
| 无 | `--version` | 显示版本 |
| `-?` | `--help` | 显示帮助 |

### 6.2 参数解析规则

- 支持 `-p` 后不带值时，通过终端无回显读取密码。
- 支持官方风格的 `-pPASSWORD`，不支持容易产生歧义的 `-p PASSWORD`。
- 命令行参数优先于配置文件。
- 首版读取配置文件中的 `[client]` 和 `[mysqlcli]` 分组。
- 密码不得写入日志、错误上下文或历史文件。
- `MYSQL_PWD` 可作为兼容输入，但必须在帮助中提示其安全风险。

### 6.3 连接行为

- 指定 `--socket` 或 `--protocol=socket` 时使用 Unix Socket。
- 指定主机或 `--protocol=tcp` 时使用 TCP。
- 连接成功后显示服务器版本、连接 ID、当前用户和当前数据库。
- 连接断开时不自动重放写操作；只提供显式重连命令。

## 7. 交互式终端设计

### 7.1 REPL 能力

- 支持单行和多行 SQL。
- 识别字符串、反引号、转义符和 SQL 注释中的分号。
- 支持 `;`、`\g` 执行和 `\G` 纵向执行。
- 支持 MySQL `DELIMITER` 语句，以便创建过程、函数和触发器。
- 支持方向键、光标编辑和本地历史记录。
- `Ctrl+C` 首次取消当前输入或查询，再次操作不应误退出整个客户端。
- `Ctrl+D` 在空输入时退出。

### 7.2 提示符

默认提示符包含当前用户、主机和数据库：

```text
root@127.0.0.1/app> 
                 -> 
```

多行输入使用次级提示符。提示符内容后续允许通过配置文件调整。

### 7.3 历史记录

- 默认文件为用户配置目录下的 `mysqlcli/history`。
- 文件权限限制为仅当前用户可读写。
- 包含明显密码操作的语句默认不写入历史，例如 `CREATE USER ... IDENTIFIED BY`、`ALTER USER ... IDENTIFIED BY` 和 `SET PASSWORD`。
- 提供关闭历史记录的配置项。

## 8. 快捷命令设计

### 8.1 命令解析原则

- 快捷命令仅在一行开头、且当前没有未完成 SQL 时识别。
- 命令名称不区分大小写，参数中的对象名称保持原样。
- 支持带引号的参数，例如 `\d "Order Detail"`。
- 快捷命令执行的元数据 SQL 使用参数化或安全标识符处理，禁止直接拼接不可信输入。

### 8.2 冲突策略

`psql` 与官方 `mysql` 的部分短命令语义冲突。项目采用以下规则：

- 用户明确需要的 `\d` 用于描述数据库对象；修改分隔符使用完整的 `DELIMITER` 语句。
- 保留官方 `mysql` 的 `\c` 清空当前输入，不将其用于切换数据库。
- 切换数据库使用 `\connect`、`\use` 或 SQL `USE database`。
- 保留 `\g`、`\G`、`\q`、`\s` 等常用官方行为。
- 所有增强命令提供可读的长名称，减少单字母命令歧义。

### 8.3 数据库对象命令

| 命令 | 说明 |
| --- | --- |
| `\l [pattern]` | 列出当前用户可见的数据库 |
| `\connect <database>` | 切换当前数据库 |
| `\use <database>` | `\connect` 的别名 |
| `\d` | 列出当前数据库的表和视图 |
| `\d <object>` | 描述表或视图的表信息、字段、索引、外键及属性 |
| `\d <pattern>` | 未精确命中对象时按包含匹配搜索对象 |
| `\desc <table>` | `\d <table>` 的别名 |
| `\describe <table>` | `\d <table>` 的别名 |
| `\ddl <table>` | 输出 `SHOW CREATE TABLE` 结果 |
| `\dt [pattern]` | 列出表 |
| `\dv [pattern]` | 列出视图 |
| `\di [table]` | 查看索引 |
| `\df [pattern]` | 查看存储函数和存储过程 |
| `\triggers [pattern]` | 查看触发器 |
| `\events [pattern]` | 查看事件调度器中的事件 |
| `\tableinfo <table>` | 查看表引擎、行格式、空间、自增值、排序规则和注释 |
| `\tablesize [pattern]` | 按数据和索引空间占用排序查看表 |

模式匹配默认支持 SQL `%` 和 `_` 通配符；未提供通配符时按包含匹配处理。

### 8.4 会话命令

| 命令 | 说明 |
| --- | --- |
| `\sessions` | 查看非空闲活跃会话 |
| `\sessions --all` | 查看全部会话，包括 `Sleep` |
| `\sessions --min-seconds <n>` | 过滤运行时间不足 `n` 秒的会话 |
| `\sessions --user <name>` | 按数据库用户过滤 |
| `\ps` | `\sessions` 的简写别名 |
| `\kill <connectionId>` | 终止指定连接当前正在执行的查询 |
| `\kill --connection <connectionId>` | 终止指定连接 |

会话输出至少包含连接 ID、用户、来源地址、数据库、命令、状态、持续时间和 SQL 摘要。默认隐藏当前 `mysqlcli` 自身会话，并截断超长 SQL。

### 8.5 锁等待命令

| 命令 | 说明 |
| --- | --- |
| `\locks` | 查看当前锁等待关系 |
| `\locks --all` | 同时展示持锁但暂未造成等待的事务 |
| `\locks --tree` | 按阻塞链展示阻塞者和被阻塞者 |

输出包含等待时长、等待连接、阻塞连接、事务 ID、对象、锁类型、锁模式及 SQL 摘要。

版本适配策略：

- MySQL 8.x 优先读取 `performance_schema.data_locks` 和 `data_lock_waits`。
- MySQL 5.7 回退到 `information_schema.innodb_locks`、`innodb_lock_waits` 和 `innodb_trx`。
- 回退前必须通过 `information_schema.TABLES` 做能力探测，避免 MySQL 8.x 中 `INNODB_LOCKS/INNODB_LOCK_WAITS` 已移除时误报旧表错误。
- 元数据表不可用或权限不足时，显示明确原因和建议权限，不把“无权限”误报为“无锁等待”。

### 8.6 主从复制命令

| 命令 | 说明 |
| --- | --- |
| `\repl` | 展示当前实例复制摘要 |
| `\repl status` | 展示复制详细状态 |
| `\repl channels` | 展示所有复制通道 |
| `\repl errors` | 仅展示复制错误和异常线程 |
| `\repl source` | 展示当前实例作为源库时的二进制日志信息 |

输出关注复制通道、源库地址、IO/SQL 线程状态、延迟、GTID、日志位点和最近错误。

兼容策略：

- 优先使用新版 `SHOW REPLICA STATUS`，必要时回退到 `SHOW SLAVE STATUS`。
- 多通道复制按通道分别展示。
- “无复制配置”“权限不足”“复制停止”必须显示为不同状态。
- MariaDB 字段差异通过版本能力探测处理，不仅依赖版本字符串。

### 8.7 用户与权限命令

| 命令 | 说明 |
| --- | --- |
| `\du [pattern]` | 查看用户列表和基本状态 |
| `\user <user@host>` | 查看单个用户详情 |
| `\grants [user@host]` | 格式化展示 `SHOW GRANTS`，省略参数时查看当前用户 |
| `\roles [user@host]` | 查看角色、默认角色和继承关系 |
| `\whoami` | 显示 `USER()`、`CURRENT_USER()` 及当前角色 |
| `\privileges` | 查看服务器支持的权限类型 |

安全约束：

- 不读取或展示 `mysql.user` 中的认证字符串、密码哈希及私钥信息。
- MySQL 5.7 不支持角色时，`\roles` 返回清晰的能力提示。
- 查看其他用户权限失败时，保留当前连接并提示最小所需权限。
- `user@host` 参数支持 `user`、`user@host` 和带引号形式；省略 host 时不得武断选中多个同名账户。

### 8.8 通用命令

| 命令 | 说明 |
| --- | --- |
| `\x [on|off]` | 切换纵向输出 |
| `\timing [on|off]` | 切换执行耗时显示 |
| `\pager [command|off]` | 设置分页器 |
| `\i <file>` 或 `\. <file>` | 在当前会话中执行 SQL 文件 |
| `\status` 或 `\s` | 查看连接状态和轻量健康快照 |
| `\reconnect` | 显式重新连接 |
| `\c` | 清空当前输入缓冲区 |
| `\p` 或 `\show` | 打印当前输入缓冲区 |
| `\e` 或 `\edit` | 调用外部编辑器编辑当前输入缓冲区 |
| `\?` | 查看快捷命令帮助 |
| `\h [command]` | 查看 SQL 或快捷命令帮助 |
| `\q` | 退出客户端 |

### 8.9 DBA 诊断辅助命令

| 命令 | 说明 |
| --- | --- |
| `\variables [pattern]` | 查看全局服务器变量，未带通配符时按包含匹配 |
| `\variables --session [pattern]` | 查看当前会话变量 |
| `\warnings` | 执行 `SHOW WARNINGS`，查看最近 SQL 告警 |
| `\W` | 切换每条 SQL 执行后自动显示告警 |
| `\innodb` | 从 `SHOW ENGINE INNODB STATUS` 提取 InnoDB 诊断摘要 |
| `\slowlog [N]` | 查看慢日志配置，带 N 时展示最近 N 条 `mysql.slow_log` |
| `\binlog` | 展示二进制日志文件和大小 |
| `\deadlocks` | 提取最近一次 InnoDB 死锁信息 |
| `\charset` | 展示常用字符集和排序规则变量 |

`\status` 的健康快照来自 `performance_schema.global_status`，仅展示低成本汇总指标，包括运行时间、请求数、慢查询数、打开表数量、连接线程数和每秒查询数。元数据权限不足时，连接状态仍应正常展示，并给出健康快照不可用的原因。

半结构化诊断命令遵循“摘要优先，保留原始入口”的原则：`\innodb` 只展示高频指标，完整原文仍可通过 `SHOW ENGINE INNODB STATUS\G` 获取；`\deadlocks` 对超长死锁段落做截断，避免一次输出淹没终端。

### 8.10 自定义快捷命令

客户端支持在 `[command.<name>]` 配置分组中定义自定义快捷命令。自定义命令优先于内置命令，因此用户可以按个人习惯覆盖内置语义，例如把 `\du <table>` 配置成查看表空间大小。

示例：

```ini
[command.du]
description = 查看表空间
sql = SELECT TABLE_NAME AS TableName, ENGINE AS Engine, ROUND((DATA_LENGTH + INDEX_LENGTH) / 1024 / 1024, 2) AS TotalMB FROM information_schema.TABLES WHERE TABLE_SCHEMA = DATABASE() AND TABLE_NAME = ?
```

自定义 SQL 中的 `?` 按快捷命令参数顺序绑定，参数数量必须与占位符数量一致。占位符识别会忽略字符串、反引号和注释中的问号。

为保护高频内置入口，`\du` 无参数始终保留为内置用户列表命令；自定义 `du` 只在带参数时覆盖内置语义。

## 9. 输出设计

### 9.1 交互输出

- 默认使用带边框的表格。
- 数值右对齐，文本左对齐，短状态字段居中。
- `NULL` 使用可配置的醒目标记，避免与空字符串混淆。
- 超长字段根据终端宽度截断，并提示存在截断。
- `\G` 或 `\x on` 使用纵向记录展示。
- 查询结束后显示行数、警告数和耗时。

### 9.2 脚本输出

- `--batch` 输出制表符分隔内容。
- `--skip-column-names` 省略表头。
- 标准输出只承载查询结果，诊断和错误写入标准错误。
- 非交互执行任一语句失败时返回非零退出码。
- 后续版本可增加 `--format=json|csv`，首版不作为兼容承诺。

## 10. 系统架构

```text
cmd/mysqlcli
    |
    +-- config          参数、配置文件和优先级合并
    +-- connection      连接建立、能力探测和重连
    +-- repl            终端交互、输入缓冲和历史
    +-- sqlparser       语句边界、引号、注释和分隔符识别
    +-- metacommand     快捷命令解析、注册和执行
    +-- diagnostics     会话、锁、复制、用户和权限查询
    +-- output          表格、纵向和批处理渲染
    +-- app             生命周期、信号和退出码协调
```

建议目录结构：

```text
cmd/mysqlcli/
internal/app/
internal/config/
internal/connection/
internal/repl/
internal/sqlparser/
internal/metacommand/
internal/diagnostics/
internal/output/
tests/integration/
docs/
```

### 10.1 核心接口边界

- `ConnectionManager`：建立连接、查询服务器能力、切换数据库和重连。
- `StatementBuffer`：维护多行输入并判断语句是否完整。
- `MetaCommandRegistry`：注册、查找和执行快捷命令。
- `QueryExecutor`：统一执行 SQL，返回结果集、影响行数、警告和耗时。
- `Renderer`：将结果渲染为表格、纵向或批处理格式。
- `CapabilityDetector`：检测版本、元数据表、复制语法和角色能力。

## 11. 版本兼容策略

能力判断应采用“功能探测优先、版本判断辅助”的原则：

1. 连接后读取服务器版本及版本注释。
2. 探测需要的 `performance_schema` 或 `information_schema` 对象是否存在。
3. 为锁、复制、角色等功能选择对应查询实现。
4. 缓存当前连接的能力集合。
5. 切换到不同服务器或重连后重新探测。

所有诊断命令必须区分：无数据、功能不支持、对象不可用和权限不足。

## 12. 安全设计

- 密码输入默认无回显。
- 日志及错误信息中的 DSN 必须脱敏。
- 历史记录过滤明显含凭据的 SQL。
- 配置文件权限过宽时给出警告。
- `\kill` 等有副作用命令显示目标连接，并要求有效的纯数字连接 ID。
- 元数据对象名必须经过安全引用，普通值必须使用参数绑定。
- 默认不开启调试日志；调试模式仍不得记录密码和 TLS 私钥。
- 不展示认证哈希等可用于离线攻击的信息。

## 13. 错误处理

- MySQL 错误展示错误码、SQLSTATE 和简洁消息。
- 语法错误不得清空尚可编辑的输入，用户可选择继续修改。
- 网络断开后提示显式执行 `\reconnect`，不自动重放 SQL。
- 权限不足时指出失败的功能和可能所需权限，但不自动执行授权。
- `Ctrl+C` 取消查询时，尽可能使用独立控制连接终止服务端查询；不可行时明确说明客户端仅停止等待。

## 14. 配置设计

配置来源按优先级从低到高合并：

1. 内置默认值。
2. MySQL 客户端配置文件的 `[client]` 分组。
3. `mysqlcli` 配置文件或 `[mysqlcli]` 分组。
4. 环境变量。
5. 命令行参数。

`mysqlcli` 专属配置包括提示符、颜色、表格样式、历史开关、历史路径、分页器、超长字段限制和默认耗时显示。

## 15. 测试策略

### 15.1 单元测试

- `-p`、短参数组合及配置优先级解析。
- SQL 字符串、注释、存储过程和 `DELIMITER` 的边界判断。
- 快捷命令参数和带引号对象名解析。
- 表格宽度、`NULL`、Unicode 和二进制字段渲染。
- 敏感 SQL 历史过滤和 DSN 脱敏。
- 服务器能力探测结果到查询实现的映射。

### 15.2 集成测试

## 18. 主要风险

| 等级 | 风险 | 应对措施 |
| --- | --- | --- |
| 高 | “兼容 mysql”范围无限扩张 | 维护明确的参数兼容矩阵，按版本迭代 |
| 高 | SQL 边界判断错误导致语句提前执行 | 对引号、注释和 `DELIMITER` 建立完整测试集 |
| 中 | MySQL 版本和 MariaDB 元数据差异 | 功能探测优先，并隔离版本查询实现 |
| 中 | 诊断命令所需权限较高 | 区分无数据与无权限，提供最小权限提示 |
| 中 | 取消查询可能需要第二连接 | 设计控制连接并明确失败行为 |
| 中 | 历史记录泄漏凭据 | 敏感语句过滤、权限约束和可关闭历史 |
| 低 | 宽字符导致表格错位 | 使用终端显示宽度计算并加入 Unicode 测试 |
